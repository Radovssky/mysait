## КЕЙС 1 — Голосовой агент для бухгалтерского консалтинга

### Заголовок
```
Голосовой агент для бухгалтерского консалтинга
```

### Slug
```
voice-agent-consulting
```

### Краткое описание
```
Голосовой AI-агент обзванивает холодную базу, квалифицирует лидов и записывает их на консультацию с живым менеджером. 250 звонков в день, без выходных и выгорания.
```

### Категория
```
voice_agent
```

### Клиент / индустрия
```
Бухгалтерский и налоговый консалтинг
```

### Контекст / задача (Markdown)
```markdown
## Что было

Компания работает в сфере бухгалтерского и налогового консалтинга. Привлечение клиентов идёт через холодный обзвон базы — потенциальных предпринимателей и компаний, которым может быть актуальна услуга.

До этого обзванивали менеджеры:

- одинаковый скрипт повторяли по сотне раз в день,
- к концу недели голос и тон «садились» — возражения отрабатывались всё хуже,
- текучка на этой роли постоянная: люди выгорают за пару месяцев,
- стоимость одного нанятого менеджера + обучение + ФОТ — это бюджет, в который никогда не помещается реальная скорость прозвона.

В итоге воронка наверху проседала: база есть, а руки до неё не доходят.

## Что должен делать агент

Брать на себя первую линию — холодный звонок: представиться, выяснить релевантность контакта, отработать стандартные возражения и, если интерес есть, записать на консультацию с живым менеджером. Менеджер должен получать уже подогретого лида с назначенным временем, а не «номер из базы».
```

### Решение (Markdown)
```markdown
## Архитектура

Агент построен как голосовая связка между базой контактов, LLM и календарём менеджера. Звонок идёт через телефонию, распознавание и синтез речи — через ElevenLabs, логика разговора — на n8n + LLM.

```text
База контактов
      ↓
n8n · постановка звонков в очередь
      ↓
Телефония · исходящий вызов
      ↓
ElevenLabs · голос + распознавание речи
      ↓
LLM · скрипт, ветвление, отработка возражений
      ↓
n8n · фиксация результата
      ↓
Календарь менеджера · запись на консультацию
```

## Что делает агент

1. **Звонит по базе** — берёт контакт из очереди, набирает номер, ждёт поднятие трубки.
2. **Ведёт разговор по скрипту** — представляется, объясняет цель звонка, задаёт квалифицирующие вопросы (тип бизнеса, текущая бухгалтерия, актуальность услуги).
3. **Отрабатывает возражения** — «дорого», «уже есть бухгалтер», «сейчас не до этого» — по подготовленному скелету скриптов, с учётом контекста разговора.
4. **Закрывает на консультацию** — если контакт квалифицирован, предлагает конкретное время и записывает в календарь менеджера.
5. **Фиксирует результат** — каждый звонок попадает в таблицу: статус (записан / отказ / перезвонить / не дозвонились), краткая суть разговора, причина отказа.
```

### Стек / интеграции
```
n8n, ElevenLabs, OpenAI, Телефония, Google Sheets
```

### Результат / метрики (Markdown)
```markdown
- 250 номеров в день — стабильно, без перерывов на обед, без «плохих дней» и без выходных.
- Скрипт и тон одинаковые от первого звонка до последнего — конверсия на холодной базе перестала зависеть от настроения менеджера.
- Менеджеры освобождены от холодных звонков и работают только с записанными на консультацию лидами.
- Стоимость одного контакта снизилась — обзвон базы больше не упирается в найм и удержание операторов.
```

## КЕЙС 2 — Telegram-бот с раскладами Таро по подписке

### Заголовок
```
Telegram-бот с раскладами Таро по подписке
```

### Slug
```
tarot-subscription-bot
```

### Краткое описание
```
AI-бот в Telegram делает персональные расклады Таро по запросу пользователя. Работает по подписке, ведёт пользователя от бесплатного расклада до платной личной консультации с экспертом.
```

### Категория
```
ai_agent
```

### Клиент / индустрия
```
Эксперт-таролог, личный бренд
```

### Контекст / задача (Markdown)
```markdown
## Что было

Эксперт-таролог работает с клиентами через личные консультации. Поток заявок шёл из соцсетей вручную: подписчик пишет в личку → договариваются о времени → проводят сессию.

Проблем было две:

- большая часть подписчиков «остывала» между «интересно» и «давайте запишемся» — пока эксперт отвечал, человек уже терял импульс,
- много запросов приходило на простые расклады, которые эксперт не хотел брать в работу лично, но и терять этих людей не хотелось — они потенциальные клиенты на полноценную консультацию.

Нужен был способ обрабатывать массовый интерес автоматически и при этом не убивать ощущение «личного» контакта с экспертом.

## Что должен делать бот

Заменить эксперта на первой линии: проводить пользователя через расклад, удерживать в воронке через подписку и аккуратно подводить тех, кому нужно глубже, к платной личной консультации.
```

### Решение (Markdown)
```markdown
## Архитектура

Бот собран на n8n с подключением LLM. Промпты для каждой категории расклада подготовлены отдельно — под семью, любовь, деньги, здоровье и т.д. — чтобы ответы попадали в стилистику эксперта, а не звучали как «ChatGPT в общем».

```text
Telegram · команда от пользователя
      ↓
n8n · проверка подписки и лимитов
      ↓
LLM · расклад по выбранной категории
      ↓
Telegram · ответ + предложение следующего шага
      ↓
n8n · триггер на консультацию (если уместно)
```

## Что делает агент

1. **Встречает пользователя** — короткое знакомство, выбор категории расклада: семья, любовь, деньги, здоровье и другие.
2. **Делает расклад** — на основе вопроса пользователя и выбранной категории генерирует расклад в стилистике эксперта, с трактовкой каждой карты и общим выводом.
3. **Управляет доступом** — бесплатный расклад в день для всех, расширенные расклады и история — по подписке. Биллинг и проверка статуса подписки — внутри бота.
4. **Закрывает на консультацию** — если запрос пользователя выходит за рамки автоматического расклада (сложная ситуация, повторяющиеся темы, явный кризис), бот предлагает записаться на личную консультацию с экспертом и передаёт лид.
5. **Удерживает в воронке** — напоминает о неиспользованных раскладах, присылает тематические подборки, не даёт пользователю «забыть» про бот после первого касания.
```

### Стек / интеграции
```
n8n, OpenAI, Telegram Bot API, Supabase
```

### Результат / метрики (Markdown)
```markdown
- Эксперт перестал быть узким горлышком — расклады для широкой аудитории идут без её участия 24/7.
- Появился новый канал монетизации — подписка как продукт, который раньше существовать не мог: эксперт физически не мог делать столько раскладов вручную.
- Личные консультации стали премиум-уровнем воронки — эксперт работает только с теми, кому действительно нужно глубже, а не отвечает на одинаковые вопросы.
- Каждый подписчик остаётся в контакте с экспертом каждый день, а не «один раз пришёл и ушёл».
```

## КЕЙС 3 — Бот для прогрева комментариев в Telegram-канале

### Заголовок
```
Бот для прогрева комментариев в Telegram-канале
```

### Slug
```
tg-comments-warmup-bot
```

### Краткое описание
```
AI-бот автоматически отвечает на комментарии под постами в Telegram-канале эксперта, ведёт диалог в обсуждении и переводит заинтересованных читателей в личку на консультацию.
```

### Категория
```
ai_agent
```

### Клиент / индустрия
```
Эксперт с авторским Telegram-каналом
```

### Контекст / задача (Markdown)
```markdown
## Что было

У клиента активный Telegram-канал: посты собирают десятки и сотни комментариев. Каждый коммент — это потенциальный лид, человек, который уже читает канал и заинтересован в теме.

Что происходило раньше:

- автор канала физически не успевал отвечать на все комментарии,
- большинство веток умирали после одного сообщения — без ответа интерес угасал,
- комментаторы, которым была нужна услуга прямо сейчас, уходили молча — никто не подсказал им следующий шаг,
- ручной прогрев каждого комментатора занимал больше времени, чем сам контент.

В итоге канал работал как медиа, а не как воронка.

## Что должен делать бот

Подключаться к комментариям как «голос автора»: поддерживать диалог в обсуждении под постом, разогревать интерес, и тем, кто проявляет реальную заинтересованность, предлагать личную консультацию — уже в личке.
```

### Решение (Markdown)
```markdown
## Архитектура

Бот подключён к Telegram-каналу клиента и работает в группе обсуждений. Реагирует на новые комментарии, классифицирует их через LLM и решает, что делать: ответить в ветке, пропустить, или перевести диалог в личку.

```text
Telegram · новый комментарий
      ↓
n8n · webhook на сообщение в группе обсуждений
      ↓
LLM · классификация: вопрос / возражение / благодарность / шум
      ↓
LLM · ответ в стилистике автора
      ↓
Telegram · публикация ответа в ветке
      ↓
n8n · триггер «перевести в личку», если уместно
```

## Что делает агент

1. **Читает все комментарии** — webhook ловит каждое новое сообщение под постом и передаёт его в n8n.
2. **Классифицирует** — LLM определяет, что это: вопрос по теме, возражение, эмоциональная реакция или шум, который не требует ответа.
3. **Отвечает в стиле автора** — даёт развёрнутый ответ с учётом контекста поста и предыдущих сообщений в ветке. Тон сдержанный, без шаблонных «спасибо за ваш вопрос!».
4. **Поднимает интерес** — задаёт встречные вопросы, делится дополнительными фактами, продолжает диалог естественно — чтобы ветка жила, а не закрывалась после одного ответа.
5. **Переводит в личку** — когда комментатор формулирует личный запрос, бот аккуратно предлагает разобрать ситуацию в личной консультации и оставляет ссылку на эксперта.
```

### Стек / интеграции
```
n8n, OpenAI, Telegram Bot API
```

### Результат / метрики (Markdown)
```markdown
- Все комментарии получают ответ — без зависимости от того, в эфире автор или нет.
- Ветки под постами стали живыми — обсуждения идут, и это поднимает охваты канала через алгоритмы Telegram.
- Появился стабильный поток заявок из комментариев в личку — раньше этот канал был «дырявым», теперь работает как полноценная воронка.
- Автор канала сосредоточен на контенте и личных консультациях, а не на ручной модерации обсуждений.
```

## КЕЙС 4 — Telegram-бот для генерации видео через Veo 3

### Заголовок
```
Telegram-бот для генерации видео через Veo 3
```

### Slug
```
veo3-video-bot
```

### Краткое описание
```
Telegram-бот превращает короткое описание от пользователя в готовый видеоролик через Veo 3. От текстового запроса до отправленного видео — без выхода из мессенджера.
```

### Категория
```
content_agent
```

### Клиент / индустрия
```
Контент-продакшн, личный бренд
```

### Контекст / задача (Markdown)
```markdown
## Что было

Клиенту нужен был быстрый способ генерировать короткие видеоролики — для соцсетей, рекламы, тестов идей. Veo 3 даёт качественный результат, но работа с ним напрямую — это:

- отдельный аккаунт и интерфейс,
- ручная подготовка промптов,
- ожидание генерации и скачивание файлов,
- никакой истории и удобства повторного использования.

Каждый ролик превращался в отдельный мини-проект на 15–20 минут — слишком дорого по времени, чтобы тестировать идеи быстро.

## Что должен делать бот

Свернуть весь процесс «идея → готовое видео» в один Telegram-чат: пользователь пишет короткий запрос — получает видео.
```

### Решение (Markdown)
```markdown
## Архитектура

Бот связывает Telegram, n8n и API Veo 3. Внутри — промпт-инжиниринг: запрос пользователя обогащается шаблоном (стиль, длительность, ракурс), отправляется в Veo 3, готовое видео возвращается в чат.

```text
Telegram · запрос от пользователя
      ↓
n8n · webhook + обогащение промпта
      ↓
Veo 3 API · генерация видео
      ↓
n8n · ожидание + скачивание файла
      ↓
Telegram · отправка видео в чат
```

## Что делает агент

1. **Принимает запрос** — пользователь пишет короткое описание ролика на естественном языке: «кот в скафандре идёт по луне, замедленная съёмка».
2. **Обогащает промпт** — добавляет технические параметры (стиль, длительность, тип кадра), которые Veo 3 ждёт для качественного результата.
3. **Отправляет в Veo 3** — запускает генерацию через API и отслеживает статус.
4. **Возвращает видео** — когда ролик готов, скачивает его и присылает в чат пользователю.
5. **Хранит историю** — все запросы и сгенерированные ролики сохраняются, можно вернуться к предыдущим идеям и сделать вариации.
```

### Стек / интеграции
```
n8n, Veo 3 API, Telegram Bot API
```

### Результат / метрики (Markdown)
```markdown
- Время от идеи до готового ролика сократилось с 15–20 минут до нескольких минут ожидания в чате.
- Тестировать креативные идеи стало дёшево — короткий промпт вместо отдельной сессии работы с Veo 3.
- Все ролики и промпты хранятся в одном месте — можно переиспользовать удачные находки и итерировать.
- Бот работает с любого устройства, где есть Telegram, — никаких отдельных приложений и аккаунтов.
```

## КЕЙС 5 — Контент-агент для кулинарного блогера

### Заголовок
```
Контент-агент для кулинарного блогера
```

### Slug
```
cooking-blog-content-agent
```

### Краткое описание
```
Автоматизация для кулинарного блогера: на основе ссылок на статьи с европейских кулинарных сайтов агент пишет авторскую статью на немецком и генерирует к ней изображения. Блогер контролирует, какие источники брать в работу.
```

### Категория
```
content_agent
```

### Клиент / индустрия
```
Кулинарный блог, Германия
```

### Контекст / задача (Markdown)
```markdown
## Что было

Блогер ведёт кулинарный сайт на немецком. Источник идей — кулинарные сайты Польши, Чехии и других стран Восточной Европы: там часто появляются интересные рецепты, которых ещё нет в немецкоязычном сегменте.

Процесс был ручным:

- блогер сам находил нужные статьи (это его принципиальное требование — выбор источника он не передаёт никому),
- переводил, переписывал в авторской стилистике,
- искал или генерировал изображения отдельно,
- собирал готовый материал в редакторе блога.

На один пост уходило несколько часов. Контент-план буксовал.

## Что должен делать агент

Забрать у блогера всё, кроме выбора источника. Блогер скидывает ссылку на статью — на выходе получает готовый авторский текст на немецком и изображения к нему.
```

### Решение (Markdown)
```markdown
## Архитектура

Триггер — сообщение блогера со ссылкой. Дальше n8n тянет содержимое страницы, отправляет в LLM на рерайт под заданный стиль, параллельно генерирует изображения и собирает готовый материал.

```text
Блогер · ссылка на статью-источник
      ↓
n8n · парсинг страницы
      ↓
LLM · перевод + рерайт в авторской стилистике
      ↓
Image API · генерация изображений по тематике
      ↓
n8n · сборка готового поста
      ↓
Блогер · готовый материал на проверку
```

## Что делает агент

1. **Принимает ссылку** — блогер сам отбирает источник и передаёт URL в систему.
2. **Парсит исходник** — извлекает текст рецепта, ингредиенты, шаги приготовления, особенности.
3. **Пишет авторский текст** — переводит на немецкий и переписывает под стилистику блога: подача, ритм, типичные обороты автора. Не дословный перевод, а собственная статья по мотивам.
4. **Генерирует изображения** — по описанию блюда и шагов создаёт картинки, которые подходят к тексту: финальное блюдо, ключевые этапы.
5. **Отдаёт готовый материал** — блогер получает текст + картинки в одном пакете. Остаётся прочитать, при необходимости поправить и опубликовать.
```

### Стек / интеграции
```
n8n, OpenAI, Image generation API, Web scraping
```

### Результат / метрики (Markdown)
```markdown
- Время на один пост сократилось с нескольких часов до проверки готового материала.
- Контент-план разогнался — частота публикаций ограничена только тем, как быстро блогер находит интересные источники.
- Авторский голос сохранился — статьи не выглядят как машинный перевод, читатели не чувствуют разницы с прежними постами.
- Блогер занимается тем, что любит и умеет лучше всего — поиском и отбором рецептов, — а не рутиной перевода и работы с картинками.
```

## Что я не стал придумывать

В кейсах нет «процентов конверсии», «X млн рублей выручки» и подобных цифр — потому что вы их не давали, а выдумывать их под продажу нельзя. Это убьёт доверие, как только клиент задаст уточняющий вопрос.

Где есть конкретика (250 звонков в день в кейсе 1) — она ваша реальная.

Если по любому из кейсов вспомните дополнительные цифры или детали (сколько комментариев в день обрабатывает бот №3, сколько подписчиков у бота №2, сколько постов вышло у блогера в кейсе №5 за месяц) — добавьте в блок «Результат». Каждая такая цифра делает кейс сильнее.
